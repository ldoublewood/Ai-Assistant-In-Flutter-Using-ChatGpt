# 代码修改规范

## 修改方式
- 在代码编写过程中，可以直接修改用户的代码文件
- 必须提供修改前后的代码对比，格式如下：
  ```
    修改前：
          [原始代码]

  修改后：
        [建议的新代码]
	  ```
	  - 清楚说明修改的原因和目的
	  - 让用户自己决定是否采用建议的修改

## 建议格式
- 使用清晰的标题说明修改内容
- 提供完整的代码上下文
- 解释修改的技术原理
- 说明修改带来的好处或解决的问题


## 代码风格
- 注意修改和新增代码需与现有的其它代码风格和架构保持一致

## 修改总结文档
- 修改总结文档存放到.enter/kiro/docs目录下

## 代码目录
- try目录下的代码为尝试／示例代码，不是本仓库正式代码的部分。除非特指，请忽略本目录下的代码。
